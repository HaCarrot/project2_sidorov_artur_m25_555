# Управление таблицами
## Создание таблицы

**Команда:** ```create <имя_таблицы> <столбец1> <тип1> [<столбец2> <тип2> ...]```

Создает новую таблицу с указанным именем и структурой. Автоматически добавляет столбец ID:int в качестве первичного ключа.

Допустимые типы данных:
* ```int``` - целые числа
* ```str``` - строки
* ```bool``` - логические значения (true/false)

## Просмотр списка таблиц

**Команда:** ```list```

Выводит список всех таблиц в базе данных с их структурой.

## Удаление таблицы

**Команда:** ```drop <имя_таблицы>```

Удаляет указанную таблицу из базы данных.

# CRUD - операции

## Вставка данных (INSERT)

**Команда:** ```insert into <имя_таблицы> values (<значение1>, <значение2>, ...)```

Добавляет новую запись в таблицу. ID генерируется автоматически.

## Выборка данных (SELECT)

**Команда:** ```select from <имя_таблицы>```

При необходимости выборки с условием используйте команду: ```select from <имя_таблицы> where <столбец> = <значение>```

Выводит записи в таблице.

## Обновление данных (UPDATE)

**Команда:** ```update <имя_таблицы> set <столбец> = <новое_значение> where <условие>```

Обновляет записи, соответствующие условию.

## Удаление данных (DELETE)

**Команда:** ```delete from <имя_таблицы> where <условие>```

Удаляет записи, соответствующие условию.

# Декораторы и дополнительные возможности

## Обработка ошибок
Все операции с базой данных теперь защищены декоратором `@handle_db_errors`, который автоматически перехватывает:
- **KeyError** - обращение к несуществующим таблицам или столбцам
- **ValueError** - ошибки валидации типов данных
- **FileNotFoundError** - отсутствие файлов с данными

## Подтверждение опасных операций
Для предотвращения случайных удалений реализована система подтверждения:

```bash
>>> drop_table users
Вы уверены, что хотите выполнить "удаление таблицы"? [y/n]: y
Таблица "users" успешно удалена.

>>> delete from users where ID = 1
Вы уверены, что хотите выполнить "удаление записей"? [y/n]: n
Операция отменена.
```

## Замер времени выполнения
Для операций ```insert``` и ```select``` добавлен замер времени выполнения:

```bash
>>> select from users
+----+--------+-----+-----------+
| ID |  name  | age | is_active |
+----+--------+-----+-----------+
| 1  | Sergei | 29  |    True   |
+----+--------+-----+-----------+
Функция select выполнилась за 0.015 секунд
```

## Кэширование запросов

Система автоматически кэширует результаты одинаковых запросов select для повышения производительности при повторных обращениях.
